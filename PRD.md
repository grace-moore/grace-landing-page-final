# Product Requirements Document (PRD)
## Personal Landing Page – Grace Moore

---

## 1. Overview

This project is a personal landing page designed to showcase Grace Moore’s professional background, skills, and experience. The site will serve as a digital portfolio and central hub linking to her resume, LinkedIn, and GitHub.

---

## 2. Goals

- Create a clean, professional online presence
- Showcase resume, skills, and experience
- Provide quick access to LinkedIn and GitHub
- Be simple, fast, and easy to navigate
- Deploy publicly using Azure Static Web Apps

---

## 3. Target Users

- Recruiters
- Hiring managers
- Professors
- Professional connections

---

## 4. Core Features

### 4.1 Hero Section
- Name: Grace Moore
- Professional title (e.g., Student / Aspiring Product Manager)
- Headshot image
- Short introduction or tagline

---

### 4.2 About Section
- Short paragraph about background
- Education (University of Iowa)
- Interests (product management, design, etc.)

---

### 4.3 Resume Section
- Downloadable resume (PDF)
- Optional embedded preview

---

### 4.4 Projects / Experience Section
- List of key projects or coursework
- Short descriptions
- Optional links

---

### 4.5 Contact / Links Section
- LinkedIn link
- GitHub link
- Email (optional)

---

## 5. Design Requirements

- Clean, minimal, modern design
- Neutral color palette (white, beige, soft tones)
- Easy-to-read typography
- Mobile responsive
- Consistent spacing and layout

---

## 6. Technical Requirements

- Built using HTML, CSS (and optional JavaScript)
- Static website (no backend required)
- Hosted via Azure Static Web Apps
- Version controlled via GitHub
- Developed using Replit

---

## 7. File Structure
/ (root)
├── index.html
├── PRD.md
├── STANDARDS.md
├── resume.pdf
├── /images
└── headshot.jpg

---

## 8. Success Criteria

- Website loads successfully via Azure URL
- Displays all sections correctly
- Links function properly
- Resume downloads successfully
- Clean and professional appearance
